# Tradie CRM agent rules

## Core rules
- Use TypeScript only
- Keep the UI simple, clean, and mobile-friendly
- Do not add unnecessary dependencies
- Reuse existing patterns and components where possible
- Keep files and folder structure tidy

## App direction
This project is a multi-tenant Tradie CRM built in Next.js.

Main sections:
- Dashboard
- Contacts
- Pipeline
- Quotes
- Jobs

## Coding preferences
- Prefer simple readable code over clever code
- Use Next.js App Router conventions
- Use Tailwind for styling
- Keep components small and reusable
- Do not overbuild features

## Before finishing any task
- Check for obvious errors
- Run lint when relevant
- Make sure the app still loads
- Summarize what changed
